== SCHRITT 1: STPO ALL DATENSTRUKTUR ==========================================

STPO ist ein BUILT-IN TradingView Indikator (KEINE Pine Script).
Daher: NICHT via data_get_pine_lines auslesbar. Eigene Extraktion notwendig.

Interne Datenstruktur:
  Primitiv-Typ: tpos (KEIN dwglines/dwgboxes wie bei Pine)
  Pfad: _graphics._primitivesCollection.tpos.get('tpo')
    - _primitivesDataById: 449 Eintraege (einzelne TPO-Preiszellen)
    - _primitiveById:        6 Eintraege (Session-Objekte, aktuelle Sicht)

  Jede TPO-Zelle hat:
    id, priceRange (=4), firstBarTime, lastBarTime, times[]
    priceRange: 4 = FIXE Bin-Breite von 4 Preis-Punkten pro TPO-Row

5 Pane-View-Renderer:
  Renderer 0: ranges      - Session-Boundary-Boxen (4 sichtbare Sessions)
  Renderer 1: blocks      - TPO-Buchstaben-Bloecke mit rows + cells
  Renderer 2: histograms  - Volume-Histogramm (wenn aktiviert)
  Renderer 3: levels      - POC / VAH / VAL Labels + rowOrCoordinate-Index [KEY]
  Renderer 4: rows        - Erweiterte Linien (Single Prints / Naked POC etc.)

PREIS-FORMEL (validiert via priceToCoordinate + roughHitTestBox):
  price = rowOrCoordinate x 4
  Beispiel: VAH-Index 1749 -> 1749 x 4 = 6996 pts
  Genauigkeit: +/- 4 Punkte (eine Bin-Breite)
  Validierung: priceToCoordinate(6732) = 769.5 = roughHitTestBox-Bottom von Block 0 (row 1684)

SESSION-REIHENFOLGE in Renderer 3 (levels):
  Indices 0,1,2   -> Aelteste sichtbare abgeschlossene Session (links)
  Indices 3,4,5   -> 2 Sessions zurueck
  Indices 6,7,8   -> Letzte abgeschlossene Session (GESTERN) [RELEVANT]
  Indices 9,10,11 -> Aktuelle laufende Session (heute, noch offen)

  Innerhalb jeder 3er-Gruppe:
    +0: VAH (blau  rgba(41,98,255), vertAlignment=1 = top)
    +1: VAL (blau  rgba(41,98,255), vertAlignment=0 = bottom)
    +2: POC (magenta rgba(255,0,255), vertAlignment=middle)

ROH-DATEN der 4 sichtbaren Sessions (rowOrCoordinate -> preis):

== SCHRITT 4A: AUSLESBARKEIT ===================================================

1. Kann STPO POC/VAH/VAL direkt auslesen?
   JA - aber mit Custom-Extraktion:
   Pfad: renderer[3]._data.levels[i].{labelText, rowOrCoordinate}
   Formel: price = rowOrCoordinate x 4
   Session-Mapping: Gruppen (0-2)=aelt, (3-5), (6-8)=gestern, (9-11)=heute

   NEIN via data_get_pine_lines (kein dwglines-Primitiv bei Built-In)

2. Session-Trennung klar identifizierbar?
   JA: 4 Gruppen mit je 3 Levels (VAH+VAL+POC) pro Gruppe.
   Aktuell-laufende Session = letzte Gruppe (Indices 9,10,11).

3. Ist STPO akkurater/einfacher als LonesomeTheBlue?
   NEIN fuer Genauigkeit: LTB Floating-Point vs STPO +/-4 Punkte-Raster.
   BESSER fuer Zusatzwerte: Naked POC, Single Prints, Poor High/Low (s. 4B).

   FAZIT: LonesomeTheBlue via data_get_market_profile = PRIMAER-Tool.
          STPO ALL = ERGAENZEND fuer Features die LTB nicht bietet.

== SCHRITT 4C: SETTINGS-ANALYSE UND EMPFEHLUNGEN ==============================

AKTUELLE SETTINGS (aus _signature-String):
  Sessions:          All               (1 Profil pro Kalendertag)
  Custom Session:    0930-1600         (Fallback wenn Sessions=Custom)
  Timezone:          Exchange           (NYSE-Zeit ET, korrekt fuer SP500)
  Block Size:        30m               (Standard, 1 TPO = 30 Minuten)
  Row Size:          Auto              (auto = 4 Punkte/Row bei SP500)
  Value Area:        70%               (AMT-Standard nach Steidlmayer)
  Extend Singles:    TRUE              (aktiv)
  Alle anderen:      FALSE

EMPFEHLUNG 1 - Sessions auf "Market only" aendern:
  Problem:  "All" inkludiert Pre/Post-Market. AMT basiert auf RTH-Sessions
            (09:30-16:00 ET) fuer US-Indices. Pre/Post-Market-Volumen ist
            minimal und verzerrt POC/VA gegenueber dem echten Tages-POC.
  Aktion:   Settings > Sessions > "Market only"
  AMT-Grund: RTH-POC = echtes institutionelles Gleichgewicht des Tages.
             Extended-Hours-POC = irrelevant fuer Intraday-Entscheidungen.

EMPFEHLUNG 2 - Naked POC aktivieren (hoechste Prioritaet!):
  Problem:  extendPOC=false -> Naked POCs werden nicht angezeigt.
  Aktion:   Settings > "Extend POC to price intersect" -> AN (Checkbox)
  AMT-Grund: Naked POC ist der starkste Magnet in der Auction Market Theory.
             Preis kehrt mit >70% Wahrscheinlichkeit zu unbezuchten POCs zurueck.
             Derzeit fehlt dieses kritische Level im Cockpit.

EMPFEHLUNG 3 - Poor High/Low aktivieren:
  Problem:  extendPoorHigh/Low=false -> Unfertige Auktionen unsichtbar.
  Aktion:   Settings > "Extend poor high/low to price intersect" -> AN
  AMT-Grund: Poor High (kein Rejection-Tail) = Kaeufer hatten letzte Kontrolle
             = hohe WS fuer Retest oben. Poor Low = umgekehrt.

EMPFEHLUNG 4 - Value Area 70% beibehalten:
  Status: Korrekt. AMT-Standard ist 70% (1 Standardabweichung). Keine Aenderung.

NAVIGATIONS-ANLEITUNG (TradingView Desktop):
  1. Mauszeiger auf STPO-Beschriftung im Chart-Legend (oben links)
  2. Zahnrad-Icon (Settings) anklicken
  3. Tab "Inputs" > Sessions > "Market only" waehlen
  4. Scrollen zu "Extend poor high to price intersect" -> Haken setzen
  5. "Extend poor low to price intersect" -> Haken setzen
  6. "Extend POC to price intersect" -> Haken setzen
  7. "OK" klicken


== SCHRITT 4D: BIAS-VERBESSERUNG - PRIORITAETSLISTE ===========================

Welche STPO-Zusatzwerte verbessern den Cockpit-Trader Bias-Score am meisten?

PRIORITAET 1: Naked POC (extendPOC aktivieren)
  Grund: POC ist das Fundament von AMT. Ein unbezuchter Naked POC ist ein
         High-Probability Entry-Level UND Target. Fehlt derzeit komplett.
         Single Prints sind aktiv, aber Naked POC ist wichtiger.

PRIORITAET 2: Single Prints (bereits aktiv - beibehalten!)
  Aktuelle Naked Single-Print-Levels (unbesucht, volles Ruckkehr-Potenzial):
  -> 6764 pts  (naechstes kurzfristiges Target)
  -> 6804 pts  (mittleres Target)
  -> 6872 pts  (hoeher)
  -> 6920 pts  (weit oben)
  -> 6932 pts  (hoecstes sichtbares SP)
  Begr.: Markt hat diese Bereiche in einer einzigen 30-min-Periode durchquert.
         Extremes Ungleichgewicht. WS > 60% fuer Rueckkehr.

PRIORITAET 3: Poor High / Poor Low (aktivieren)
  Grund: Poor High/Low = unfertige Auktion = starkes direktionales Signal.
         Erkennt ob der Markt mit Kontrolle der Kaeufer oder Verkaeufer schloss.
         Direkte Auswirkung auf Bias-Bestimmung (Bullisch/Baerisch).

PRIORITAET 4: Volume POC (extendVolumePoc)
  Grund: Zeigt wo Tick-Volumen konzentriert war (sekundaer bei Vantage/CFD).
         Wenn Volume-POC = TPO-POC: starke Konfluenz.
         Wenn divergent: Signal fuer Preis-Volume-Divergenz.

PRIORITAET 5: Naked VAH / VAL
  Grund: Unbezuchte VA-Grenzen als sekundaere Referenzlevels.
         Weniger wichtig als POC aber gut fuer TP-Platzierung.

PRIORITAET 6: Summary Info Panel
  Grund: Wuerde direkt maschinenlesbare Werte liefern ohne Custom-Extraktion.
         Aktuell tiefste Prio da bestehende renderer[3]-Extraktion funktioniert.
